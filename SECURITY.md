# Security Policy

## Reporting a vulnerability

Do not open a public GitHub issue for security-sensitive reports.

If the repository enables GitHub Security Advisories, use a private advisory.
Otherwise, contact the maintainer directly through the repository owner contact
channel and include:

- affected version
- impact summary
- reproduction steps
- a minimal proof of concept if available

I will validate the report, work on a fix, and coordinate disclosure once a
patched version is ready.
